Project and Research Work

Implementing Prioritized Experience Replay to DQN

Reinforcement learning is a very important subset of machine learning which is highly effectiveand applicable to many real situations. Therefore, focusing on how to improve specific aspects ofthe learning process could have a profound impact on the ability and accuracy of reinforcementlearning programs. The main focus of the project is using a method called prioritized replay which is a method of changing the sampling probability of reinforcement training data based on the expected learning progress as measured by the temporal difference error which will be explained more learninglater in this paper. The benefits of this method include making better use of our memory to storemore relevant training data, allowing us to tune stochastic gradient descent methods to make them more efficient and accurate. Furthermore, it also improves upon the common naive gradient descent assumption that the training data is always independent and of a uniform distribution.

MBot Mini Project

The purpose of this project was to demonstrate programming techniques, control systems, mapping, localization, and path-finding algorithms, and engineering methods commonly used in mobile robotics for the class ROB 550 at the University of Michigan. Our platform was the MBot Mini, a mobile robotics kit consisting of two microcomputers (one for low level motor control and one for high level programming), two motors with gearboxes, wheels, and magnetic encoders, a LIDAR sensor, and a front facing camera. Throughout the project, the Lightweight Communications and Marshalling system (LCM) was used to communicate data between programs (and even between computers). First, a motor control system was created that, given a desired path and the robots current position data, could generate instantaneous velocity commands for the wheels. Then, a Simultaneous Localization and Mapping (SLAM) program was developed, allowing the robot to make use of the LIDAR sensor to map its environment and get more reliable position data as well. Finally, an A* path planning algorithm was implemented and programs were written allowing the robot to navigate around obstacles, explore its environment, and even localize on a map without knowing its starting position. Skeleton code was provided for much of the project to shift the focus away from bit bashing and onto higher level topics. Additionally, programs were provided to allow us to test the solution, most notably the botgui, that plotted useful data such as the robot position and SLAM map based on LCM messages it received.

Force Analysis of a Multi Finger Robotic Gripper

This project thesis involved the study of Jamia Hand, which is an underactuated tendon driven four- fingered robotic gripper. An analogy is then presented between the human hand and the Jamia Hand w.r.t the degrees of freedom, angle of rotation, number of joints, and the number of fingers of the different joints. A mathematical model is also shown. The friction losses that come into play between the tendon cables and the hand’s surface are calculated using the simplified Dahl’s model and the belt-pulley approach. A simple model is constructed to calculate the contact forces too. Finally, kinematic models for the motion of thumb and the fingers is shown so as to trace their motion at any point, with some MATLAB graphs showing the variation between the angle of rotation and the top distances.
Paper Code: BPA-4S-15
Organization: UGC Journal

Robotic Manipulator Project

This project was a part of the class ROB 550 at the University of Michigan. The purpose of this project was to implement forward and inverse kinematics modeling of a 6-DOF manipulator, programming it for grasping. Sensing model was developed for 3D image calibration and object detection with OpenCV. Finally, path planning and smoothing was implemented along with state machines. This project involved ROS as the mediator.
